In 2012 the development of VLFeat is sponsored by the PASCAL Harvest Programme. Many hours of coding went into the project by several new contributors. The novel developments will be made public in the next several weeks as the code is tuned and tested.

A particularly signifcant achievement is the creation of a new subroject, VLBenchmakrs, for the evaluation of feature detecrtors and descriptors. VLBenchmarks is meant to provide a future-proof benchmarking suite. The first release includes reimplementations of standard feature benchmarks to replace aging legacy code and a brand new benchmark. In the future, it will be used to deliver to the community new, modern benchmarks in a consistent and simple to use pacakge.

Version 0.9.18 (predicted January)

  • Large scale Approximate Nearest Neighbours (ANN) K-means.
  • Large scale ANN Gaussian Mixture Models (GMMs).
  • Fisher Kernel encoding.
  • Locality Linear Coding (LLC).

Version 0.9.17 (predicted end November)

  • Local Ternary Patterns (LTP).
  • Local Binary Patterns (LBP).

Version 0.9.16 (predicted early October)

  • VLBenchmarks: detector repeatability, descriptor matching score, and image retrieval benchmarks.
  • Affine covariant feature detectors. The aim is to create a future proof replacement of the legacy implementation.

Version 0.9.15 (released)

  • Histogram of Oriented Gradients (HOG) features.
  • New Stochatstic Gradient Descent (SGD) Support Vector Machine (SVM) implementation. Layed down a good infrasructure to add mroe learning algorithms in the future.
  • Several utility functions (hashed summation, integral images, plotting functions).
  • Significant cleanup of the codebase, outsdanding bugs squashed.
  • Created the VLFeat Team for a more open development.
